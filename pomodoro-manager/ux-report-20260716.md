## 二、发现问题清单（深挖细节，不与上一轮重复）

### 🔴 严重（1 个）

**Q1. 首次添加任务不自动设为当前任务——核心流程断裂**
- 位置：`src/renderer/renderer.js` addTask() + `src/main.js` task:add handler + `src/core/pomodoro-core.js` addTask()
- 现象：用户添加第一个任务后，"当前任务"仍显示"未选择任务"，必须额外点击任务行右侧的小圆点才能设为当前任务，导致番茄完成时无法自动关联任务进度。
- 影响：首次体验的核心流程断裂。新用户最自然的操作是"建任务→开始专注"，但当前需要"建任务→找小圆点→点击→开始专注"，多出一步且无明显引导。
- 修复：`task:add` IPC handler 中，若 `core.currentTaskId` 为空且新任务未完成，自动 `setCurrentTask(t.id)`。

### 🟡 中等（5 个）

**Q2. 关闭窗口无托盘提示，首次用户误以为程序退出**
- 位置：`src/main.js` mainWindow.on('close')
- 现象：首次点击关闭按钮直接隐藏到托盘，无任何提示。用户以为程序已退出，下次想用可能再启动一个实例或找不到窗口。
- 修复：首次最小化到托盘时，通过 `tray.displayBalloon()` 显示气泡通知"已最小化到托盘，点击托盘图标可恢复窗口"，仅提示一次（`trayHintShown` 标记）。

**Q3. 阶段标签点击切换丢失进度无确认**
- 位置：`src/renderer/renderer.js` phaseTabs click handler
- 现象：专注进行到一半（如剩余 15 分钟），误点"短休息"标签会立即调用 `switchPhase` 重置计时并切换阶段，丢失专注进度，无任何确认。
- 修复：当前阶段运行中且进度 >10% 时，首次点击给 toast 提示"当前阶段进度 X%，再次点击确认切换"，3 秒内再次点击同标签才真正切换。

**Q4. 设置数值越界静默修正无提示**
- 位置：`src/renderer/renderer.js` settingsSave handler
- 现象：用户在设置中输入专注时长 999 分钟，被 `clamp(v, 1, 90)` 静默修正为 90，无任何提示。用户保存后以为专注时长是 999 分钟，实际是 90，造成困惑。
- 修复：保存时对比原始值与 clamp 后值，若有差异，toast 提示具体被修正的字段（如"已保存（专注已调整为最大 90）"）。

**Q5. 删除任务无撤销，误点直接丢失**
- 位置：`src/renderer/renderer.js` taskList click delete handler
- 现象：任务行的 × 删除按钮 hover 才显示，但点击即删除，无法恢复。用户误点会丢失任务及其番茄进度记录。
- 修复：删除前缓存任务快照，删除后 toast 提示"已删除「任务名」"并提供"撤销"按钮（4.5 秒有效）。撤销时通过新增的 `addTaskRaw` API 以原始对象（保留 id/pomodoros/createdAt）恢复任务。

**Q6. 任务输入框无字符计数，临近上限无感知**
- 位置：`src/renderer/index.html` taskInput + `src/renderer/renderer.js`
- 现象：`maxlength=80` 限制输入长度，但界面上无任何字符计数显示，用户不知道还能输入多少字符，临近上限时输入突然被截断而无提示。
- 修复：在输入框旁添加 `0/80` 字符计数器，`input` 事件实时更新；达到 70 字符时计数器变橙色高亮提醒。

### 🟢 轻微（4 个）

**Q7. 今日柱状图无数据时几乎不可见**
- 位置：`src/renderer/renderer.js` renderWeekChart + `src/renderer/styles.css` .week-bar.empty
- 现象：空数据柱最小高度 `Math.max(4, h)` = 4px，在 76px 高的图表区几乎看不见，用户无法分辨"今天没数据"和"渲染异常"。
- 修复：空柱最小高度提升至 10px，样式改为虚线边框占位（`border: 1.5px dashed`），今日空柱用蓝色虚线 + 浅蓝背景，明确表达"有位置但无数据"。

**Q8. "投入 X / Y" 标签含义不清**
- 位置：`src/renderer/renderer.js` renderTasks taskFooter
- 现象：底部统计条带显示"投入 3 / 8"，"投入"一词含义模糊，用户不知道 X 是实际番茄数还是时间投入。
- 修复：改为"番茄 3 / 8"，与项目核心概念（番茄数）对齐，语义更明确。

**Q9. 设置弹层关闭按钮 × 缺 aria-label，可访问性差**
- 位置：`src/renderer/index.html` settingsClose + taskInput/taskEstimate/taskAddBtn
- 现象：× 关闭按钮、任务输入框、预估番茄数输入框、添加按钮均无 `aria-label`，屏幕阅读器无法识别用途。
- 修复：为这些元素添加语义化 `aria-label`（"关闭设置"/"任务名称"/"预计番茄数"/"添加任务"）。

**Q10. 缺少数字快捷键切换阶段**
- 位置：`src/renderer/renderer.js` keydown handler + `src/renderer/index.html` kbd-hint
- 现象：阶段切换只能鼠标点击，键盘用户无法快速切换专注/短休息/长休息。已有空格/R/S 快捷键，但缺数字键。
- 修复：新增 `1`/`2`/`3` 数字键分别切换专注/短休息/长休息；kbd-hint 提示同步更新为"空格 开始/暂停 · R 重置 · S 跳过 · 1/2/3 切换阶段"。

**新增 API**：
- `addTaskRaw(task)`：以原始对象恢复任务（撤销删除用），保留 id/pomodoros/createdAt/estimate/completed。在 `PomodoroCore` 类与 `preload.js`/`main.js` IPC 中完整贯通。

**toast 增强**：
- `toast(msg, actionLabel, actionFn)` 支持可选的操作按钮（如"撤销"），按钮 4.5 秒后随 toast 消失。新增 `.toast-action` 样式（蓝色高亮、hover 背景）。
