## 二、发现问题清单

### 🔴 严重（1 个）

**P1. 渲染层完全失效——`renderer.js` 重复声明 `const running` 致 SyntaxError**
- 位置：`src/renderer/renderer.js` `renderTimer()` 内第 149 行与第 172 行均声明 `const running`。
- 现象：`node --check` 报 `SyntaxError: Identifier 'running' has already been declared`。整个 renderer.js 无法解析执行 → 应用启动后所有按钮失效、计时器不动、统计数据为 0、无法添加任务，仅剩静态 HTML 外壳。
- 影响：**应用完全不可用**。这是本次巡检最严重的发现。

### 🟡 中等（6 个）

**P2. 阶段标签页配色为死代码**
- CSS 定义了 `.phase-tab.active.working/short_break/long_break` 三色（蓝/绿/橙），但渲染层只切换 `active` 类，从未给标签添加阶段类，导致激活标签永远是默认色，阶段配色完全不生效。

**P3. 空任务名静默失败**
- `addTask()` 在标题为空时直接 `return`，无任何提示。用户按回车无反应，不知道为何没添加成功。

**P4. 跳过提示文案误导**
- idle/paused 状态点跳过，与严格模式下休息不可跳过，统一提示"当前阶段不可跳过"。idle 时该文案令人困惑（实际是"未开始计时"）。

**P5. 设置弹层 Esc 键无效**
- 键盘处理器在设置弹层打开时直接 `return`，Esc 既不能关闭弹层，也不做任何事。违背通用键盘习惯。

**P6. 任务预估番茄数可越界**
- 添加任务时 `parseInt(el.taskEstimate.value, 10) || 1` 不强制上限，输入框虽 `max=20` 但可手动键入 999。

**P7. 白噪音自动播放静默失败**
- 启动时 `loadState()` 调用 `startWhiteNoise()`，但浏览器自动播放策略下 `AudioContext` 处于 suspended（无用户手势），白噪音静默不响，而开关显示"开"，误导用户以为正在播放。

### 🟢 轻微（3 个）

**P8. 设置弹层缺少显式"取消"按钮**
- 仅有"×"关闭，用户不确定是否丢弃修改。

**P9. 任务排序就地修改核心状态**
- `renderTasks` 中 `tasks.sort()` 直接修改 `state.tasks` 数组顺序，污染核心状态。

**P10. 暂停状态丢失阶段上下文**
- 暂停时阶段标签只显示"已暂停"，用户不知道暂停的是专注还是休息。

## 三、修复内容

| # | 问题 | 修复 | 文件 |
|---|---|---|---|
| P1 | 重复 `const running` 致白屏 | 删除第 172 行重复声明，复用上方 `running` | renderer.js |
| P2 | 阶段标签配色死代码 | HTML 给每个 tab 加阶段类，激活时三色生效 | index.html |
| P3 | 空任务名无反馈 | 空输入时 `toast('请输入任务名称')` 并聚焦 | renderer.js |
| P4 | 跳过文案误导 | 区分 idle/paused（"计时未开始"）与严格模式（"严格模式下休息不可跳过"），按钮与键盘两处 | renderer.js |
| P5 | Esc 不关设置 | 弹层打开时 Esc 关闭，其余快捷键不触发 | renderer.js |
| P6 | 预估数越界 | `clamp(est, 1, 20)` 限制 | renderer.js |
| P7 | 白噪音静默失败 | ctx 未 running 时不建音源；注册首次手势监听解锁后按需补启；`applyWhiteNoise` 统一状态 | renderer.js |
| P8 | 无取消按钮 | 弹层增加"取消"按钮 + spacer 布局 | index.html / styles.css |
| P9 | 排序污染状态 | `tasks.slice().sort()` 用副本 | renderer.js |
| P10 | 暂停丢上下文 | 暂停时显示"已暂停 · 专注/短休息/长休息" | renderer.js |

**美学打磨（基于 mimo 反馈，疯狂扣细节模式）：**
- 柱状图非今日柱改为中性灰，仅今日柱保留蓝色强调，消除配色冲突。
- 计时器区域强化为 `--shadow-lg` 焦点阴影，增加内边距留白。
- 当前任务横幅去掉渐变背景，柔化为 `surface-2`，降低视觉权重。
- 主区域 padding/grid-gap 增加，改善呼吸感。
- 进度条光晕与圆环光晕柔化，轨道色提亮。
- 任务子项进度文字加深（`--text-3` → `--text-2`）提升对比度。
- 圆环粗细经多轮校准（过细则失进度感），最终回到 track 10 / progress 12 并柔化轨道色。
