## 二、发现问题清单（深挖细节，不与上一轮重复）

### 🔴 严重（1 个）

**Q1. 阶段切换/数字键切换确认阈值回归（10% 而非 3%）——上一轮承诺未落地**
- 位置：`src/renderer/renderer.js` 第 589 行（phaseTabs click）和第 850 行（数字键 1/2/3）
- 现象：上一轮（07-17）Q3、Q4 报告明确写道"调整阈值为 `progress > 3%`（即专注 >45 秒就触发确认），更严格保护用户进度"，但代码中实际仍为 `> 0.1`（10%）。25min 制下 10% = 2.5 分钟，专注刚启动 1-2 分钟内误切标签或按数字键仍立即丢失进度，无任何确认。
- 影响：与上一轮报告承诺不一致，专注刚开始误点立即丢失进度。这是上一轮宣称已修复但实际未落地的回归 bug。
- 修复：两处 `> 0.1` 改为 `> 0.03`，真正兑现上一轮承诺。

### 🟡 中等（5 个）

**Q2. 设置弹层取消/关闭/Escape 丢失未保存改动无提示**
- 位置：`src/renderer/renderer.js` settingsMask/settingsClose/settingsCancel/Escape 处理
- 现象：用户在设置中改了专注时长 25→45、每日目标 8→12、开启严格模式，点击"取消"或右上角"×"或点击遮罩或按 Escape，所有改动直接丢失，无任何确认。改完几分钟的设置一键清零，用户无机会挽回。
- 修复：新增 `closeSettingsWithConfirm()` 函数和 `hasUnsavedSettingsChanges()` 检测器。检测到任意设置项有变化时，关闭需二次确认。首次提示"设置已修改未保存，再次关闭确认丢弃"，3 秒内再次点击才真正关闭。Escape 也走同一确认流程。

**Q3. 跳过专注无二次确认（与 reset/phase 切换不一致）**
- 位置：`src/renderer/renderer.js` skipBtn click handler
- 现象：reset 有 `confirmReset()`，phase 切换有 `dataset.confirm`，数字键有 `_numConfirm`，但 skip 按钮在 working 状态点击直接跳过，丢失剩余专注时间（可能 20+ 分钟）。虽然上一轮 Q1 已修复"跳过不计入统计"，但用户仍可能误点 skip 按钮丢失大量进度，与 reset 的"保护机制"形成不一致。
- 修复：跳过专注时（剩余时间 > 30% 时，即 progress < 0.7）需二次确认。toast 提示"将放弃剩余 N 分钟专注，再次点击确认跳过"，3 秒内再次点击才真正跳过。跳过休息不触发确认（休息跳过无进度损失）。

**Q4. 完成任务时无视觉庆祝反馈（与番茄完成反差大）**
- 位置：`src/renderer/renderer.js` taskList click handler + `src/renderer/styles.css`
- 现象：用户点击任务 radio 完成任务，UI 仅简单划掉标题（line-through + opacity 0.55），无任何视觉反馈。与番茄完成时的"celebrate"波纹动效 + 数字弹跳形成强烈反差，让用户感觉"完成一个任务"的成就感缺失，任务完成体验单薄。mimo 反馈"当前任务标签样式过于克制"。
- 修复：完成任务时给 task-item 添加 `task-celebrate` 类，触发 `taskCelebrate` 动画：scale 1→1.04→1 + 绿色光晕扩散 0.7s，呼应 macOS 完成提醒事项的勾选弹跳感。700ms 后自动移除该类。

**Q5. 严格模式开关无说明，用户难理解含义**
- 位置：`src/renderer/index.html` 严格模式 setting-row
- 现象：标签只有"严格模式（不可跳过休息）"，但实际行为是"工作完成后强制进入休息，不能跳过"。用户可能误以为是"不能暂停"或"必须完成所有番茄"，含义模糊。新用户开启后遇到无法跳过休息会困惑。
- 修复：在 label 旁加一个 14px 圆形小问号 `?` 图标（`.setting-help`），hover 显示原生 tooltip："启用后，专注结束后的休息阶段无法被跳过，强制执行番茄工作法的休息规则。保护你的休息节奏，避免连续专注导致疲劳。"问号 hover 时变蓝色 + 浅蓝背景，呼应苹果设计语言。

**Q6. 当前任务条点击无任何操作，用户期望跳转**
- 位置：`src/renderer/renderer.js` currentTask element
- 现象：当前任务条显示任务名，但点击无任何反应。用户期望点击高亮当前任务、或滚动列表到对应项。当前任务条作为视觉锚点却"看起来可点击实际不可点"，与用户直觉相悖。
- 修复：点击当前任务条时，`scrollIntoView({ behavior: 'smooth' })` 滚动任务列表到对应项，并触发 `flashHighlight` 高亮闪烁动画（1.2s）：背景从浅蓝→深蓝→浅蓝渐变，配 inset 蓝色细条 + 外发光，让用户明确感知到"当前任务在列表中的位置"。鼠标 hover 时增强视觉反馈（边框变蓝、背景渐变加深）+ pointer 光标。

### 🟢 轻微（4 个）

**Q7. 数字快捷键 1/2/3 在 kbd-hint 中缺失**
- 位置：`src/renderer/index.html` kbd-hint
- 现象：kbd-hint 显示 Space/R/S，但 1/2/3 数字快捷键切换阶段未显示，用户不知道有这个功能（即使代码已实现）。键盘党失去发现路径。
- 修复：在 kbd-hint 中追加 "1 2 3 阶段" pill（保持紧凑，3 个数字键合并在一个 group 内）。后续 mimo 反馈"信息过载"后移除了 N 新任务项，仅保留 1/2/3，平衡可发现性与视觉噪音。

**Q8. 添加任务无 toast 反馈**
- 位置：`src/renderer/renderer.js` addTask function
- 现象：添加任务成功后任务列表立即更新，但无任何提示。用户不确定是否成功，尤其在快速连续添加时无法确认每次是否生效，缺少操作确认感。
- 修复：toast 提示"已添加「任务名」"（任务名超过 12 字截断加 …），1800ms 后自动消失。与"已重置"、"已切换到「专注」"等已有 toast 形成统一反馈语言。

**Q9. 周柱状图今日为 0 时无 0 数字标注**
- 位置：`src/renderer/renderer.js` renderWeekChart
- 现象：今日柱体为空（dashed border），但 week-bar-count 文字显示为空字符串，导致今日柱体上方空白，用户难以识别"今日 0 番茄"还是"今日数据加载失败"还是"今日尚未开始"。其他日期为 0 时也空白合理（无锚点），但今日作为重要锚点应明确显示。
- 修复：今日为 0 时显示 "0" 数字标注，颜色用 `var(--accent)`（CSS 中 `.week-bar-wrap.today .week-bar-count` 已是 accent 色）突出今日锚点。其他日期仍按原逻辑（0 时不显示数字）。

**Q10. 设置弹层关闭按钮 "×" 字符样式与 SVG 图标不一致**
- 位置：`src/renderer/index.html` settingsClose button
- 现象：右上角关闭按钮用文字字符 "×"（font-size: 22px），与设置齿轮、其他按钮使用的 SVG path 风格不统一。"×" 字符在不同字体下渲染粗细不一，缺乏设计感。
- 修复：改用 SVG path `<path d="M6 6l12 12M18 6L6 18">` 实现 X 关闭图标（stroke-width 1.8，linecap round），与项目其他 SVG 图标统一风格。

## 八、结论

本轮巡检在上一轮基础上，首要修复了 **Q1 阈值回归 bug**——上一轮报告承诺将阶段切换/数字键切换确认阈值从 10% 收紧到 3%，但代码中实际仍为 10%。这是上一轮宣称已修复但实际未落地的回归问题，专注刚开始 1-2 分钟内误点立即丢失进度。本轮真正兑现承诺，两处 `> 0.1` 改为 `> 0.03`。

新增的 `closeSettingsWithConfirm` / `confirmSkipWork` / `taskCelebrate` / `flashHighlight` 共同构成更完整的"误操作保护 + 操作反馈"网络：
- 设置改动丢失保护（Q2）
- 跳过专注确认（Q3）
- 任务完成庆祝（Q4）
- 当前任务跳转高亮（Q6）

视觉打磨方面，**计时器数字字号放大 + 字重加粗**（60→68px、600→700）是 R2 评分提升的关键（计时器焦点 6.0 → 8.0，+2.0）。但 kbd-hint 加项导致信息密度下降（7.0 → 6.5），已通过移除 N 项收敛。

修复后核心测试 119/119 通过，语法检查全通过。mimo 评分从 6.5 提升至 7.0（+0.5）。mimo 新反馈的"进度表达不一致"、"任务清单滚动暗示"等问题可作为下一轮深入方向。
