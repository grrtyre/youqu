## 三、发现的问题与修复（共 22 项）

### 🔴 严重（4 项功能修复）

#### F1 · 剪贴板去重置顶
- **问题**：复制旧内容会产生重复条目累积，列表越来越乱
- **修复**：在 `startClipboardPolling` 中，检测到内容变化时先 `clipboardHistory.filter(i => i.content !== current)` 移除相同旧条目，再 `unshift` 到顶部
- **文件**：`src/main.js`

#### F2 · 删除条目支持撤销
- **问题**：误删条目无法恢复，与"剪贴板历史"的核心价值相悖
- **修复**：
  - `main.js` 新增 `restore-item` IPC handler，把删除的条目重新插回顶部；图片条目若文件已删则拒绝恢复
  - `preload.js` 新增 `restoreItem(item)` API
  - `renderer.js` 新增 `showUndoToast(item)`：删除后弹 toast「已删除 [撤销]」，停留 4.5s 可点击撤销
  - `styles.css` 新增 `.toast.undo` / `.toast-msg` / `.toast-undo` 样式
- **文件**：`src/main.js`、`src/preload.js`、`src/renderer/renderer.js`、`src/renderer/styles.css`

#### F3 · 首次启动欢迎引导
- **问题**：新用户第一次打开看不到任何提示，不知道按 `Ctrl+Shift+V` 唤起、不知道复制会自动记录
- **修复**：
  - `main.js` 新增 `welcomeShownFile` 标记文件 + `firstRun` 变量 + `get-first-run` / `mark-welcome-shown` IPC
  - `index.html` 列表上方新增 `#welcomeCard`（默认隐藏），含品牌 SVG / 标题 / 快捷键提示 / 知道了按钮
  - `renderer.js` 新增 `maybeShowWelcome()`，首次启动展示一次
  - `styles.css` 新增 `.welcome-card` / `.welcome-icon` / `.welcome-title` / `.welcome-hint` / `.welcome-ok` / `kbd` 样式
- **文件**：`src/main.js`、`src/preload.js`、`src/renderer/index.html`、`src/renderer/renderer.js`、`src/renderer/styles.css`

#### F4 · Shift+点击/Shift+Enter 仅复制不关窗
- **问题**：连续复制多条需要反复唤起面板，效率低
- **修复**：在 `listEl.click` 与键盘 `Enter` 处理中检测 `e.shiftKey`，若按下则只 `copyItem` + toast，不调用 `pasteToFront`、不 `window.close()`
- **文件**：`src/renderer/renderer.js`

### 🟡 中等（6 项视觉/交互修复）

#### V1 · 类型徽章中文化
- **问题**：原徽章显示 LINK/MAIL/CODE/TEXT/TEL 英文，与中文界面不一致
- **修复**：`TYPE_LABEL = { code:'代码', link:'链接', email:'邮箱', phone:'电话', text:'文本', image:'图片' }`
- **文件**：`src/renderer/renderer.js`

#### V2 · 品牌 emoji → 渐变 SVG
- **问题**：顶栏 📋 emoji 风格不统一，与苹果白高端感冲突
- **修复**：替换为带 `linearGradient` 的剪贴板 SVG，外套 `.brand-logo` 圆角淡蓝容器
- **文件**：`src/renderer/index.html`、`src/renderer/styles.css`

#### V3 · 空状态 SVG 插画
- **问题**：空状态用 📭/🔍 emoji，廉价感强
- **修复**：根据 `currentSearch`/`currentFilter` 切换两套 SVG 插画（搜索无结果 vs 全空），用 `url(#cbGrad)` 与品牌色统一
- **文件**：`src/renderer/renderer.js`、`src/renderer/index.html`、`src/renderer/styles.css`

#### V4 · 数据路径可点击打开文件夹
- **问题**：设置面板显示数据路径但用户无法快速定位文件
- **修复**：`main.js` 新增 `open-data-folder` IPC（调用 `shell.showItemInFolder(historyFile)`），`renderer.js` 给 `dataPathEl` 加 cursor:pointer + click 事件
- **文件**：`src/main.js`、`src/preload.js`、`src/renderer/renderer.js`

#### V5 · 搜索框白底 + 双层阴影 + focus 蓝色光环
- **问题**：mimo 指出"搜索框样式过于基础缺少品牌感融入"

#### V6 · 底栏强化（kbd 快捷键 + 计数胶囊 + 按钮阴影）
- **问题**：mimo 指出"底栏信息单薄与主体区域视觉脱节"
- **修复**：计数用胶囊样式（带边框 + 浅灰背景）；快捷键提示用 `<kbd>` 标签渲染；清空按钮加 SVG 图标 + 双层 hover 阴影；底栏背景加 backdrop-filter

## 四、误操作与边界测试

| 测试场景 | 结果 |
|---|---|
| 空输入搜索 | 正常显示"没有匹配的记录"空状态 |
| 超长字符串（>200 字符） | `truncate(s, 200)` 截断 + 省略号 |
| 快速连续点击多个条目 | 每次都触发 copyItem，无错乱 |
| 切换筛选 tab + 编辑模式 | 切换筛选时自动退出编辑模式 |
| 删除后立刻点撤销 | restore-item 正确恢复，列表正确刷新 |
| 图片条目删除（图片文件已不存在） | restore-item 返回 false，提示"恢复失败" |
| Esc 键在编辑模式 | 退出编辑不退出窗口 |
| Esc 键在确认弹窗 | 关闭弹窗不触发清空 |
| Shift+点击 | 仅复制不关窗，可连续操作 |
| 首次启动后再次启动 | welcomeShownFile 已存在，不再展示欢迎卡片 |

## 五、遗留问题（后续巡检可继续优化）

1. **代码块语法高亮**：当前代码预览仅等宽字体 + 行号，未做语法着色（如关键字/字符串/注释着色）。可考虑引入轻量 highlight.js 或自实现简易高亮。
2. **mimo 指出"卡片阴影层次仍可继续打磨"**：四层阴影已应用，但与 macOS 原生卡片相比仍有差距，可继续微调 alpha 与 blur 半径。
3. **设置面板留白**：mimo 6.5/10 指出"模态窗口内部留白不足，布局拥挤"，后续可增加设置行 padding 与面板内边距。
4. **图片缩略图大图预览**：当前预览面板 `max-height: 320px`，超大图片显示不全，可考虑点击放大查看原图。
5. **多显示器场景**：`toggleWindow` 仅定位到主屏右下角，副屏用户可能找不到窗口。
