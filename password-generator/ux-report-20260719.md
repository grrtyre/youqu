## 二、本轮新发现问题清单

### 🔴 严重（核心交互断裂）

| # | 问题 | 位置 | 现象 |
|---|------|------|------|
| P21 | 复制按钮无视觉反馈 | renderer.js `copyPwd` / `copyPhrase` / `copy-mini` | 频繁使用的核心动作，点击后仅 toast 提示，按钮本身无任何状态变化（无 ✓ 闪现、无颜色变化），用户无法即时确认"是这一个按钮触发了复制" |
| P22 | 长度滑块拖动期间密码与元信息不同步 | renderer.js `lengthSlider input` | 拖动滑块时数字输入框已变（如 16→32），但密码展示区、强度条、长度/字符池/熵/破解耗时仍显示旧密码（16 位）的数据，造成视觉错位："滑块=32，展示=16 位" |
| P23 | 字符类型勾选 / 分隔符切换不重新生成 | renderer.js `chk*` change / `seg` click | 取消勾选"符号"后展示的密码仍含符号；切换分隔符后口令不变。与滑块 `change` 自动生成行为不一致，用户以为没生效 |

### 🟡 中等（反馈缺失 / 可用性）

| # | 问题 | 位置 | 现象 |
|---|------|------|------|
| P24 | Ctrl+G 在不可生成的 Tab 静默无效 | renderer.js keydown Ctrl+G | 在强度检测 / 历史记录 Tab 按 Ctrl+G 无任何反馈，用户以为快捷键失效或软件卡死 |
| P25 | 复制按钮无 disabled 态 | index.html / styles.css | 密码未生成时，复制按钮仍可点击，仅弹"暂无内容" toast，按钮无视觉禁用态，看起来和正常状态完全一样 |
| P26 | 弱密码配置无警告 | renderer.js `genPassword` | 长度=4 + 仅数字勾选会生成 4 位纯数字密码（强度极弱），无任何提示用户当前配置不安全；用户可能误以为这是"强密码" |
| P27 | 长度 / 单词数量输入框无 Enter 确认 | renderer.js `lengthInput` / `wordInput` | 在数字输入框输入数字后按 Enter 不生成密码，必须手动点"生成密码"按钮，违反 Enter 即确认的通用约定 |
| P28 | 强度页输入框 placeholder 与上方 label 文案重复 | index.html `strengthInput` | 上方 label 显示"输入待检测密码"，下方 placeholder 又是"在此输入或粘贴密码…"，"输入"两字重复，mimo 指出冗余 |
| P29 | 强度条 bits 标签视觉对齐错位（mimo） | styles.css `.strength-info` | 强度条与右侧"83 bits"文本未垂直对齐，bits 标签飘空，与强度条不在同一视觉重心 |

### 🟢 轻微（文案 / 一致性）

| # | 问题 | 位置 | 现象 |
|---|------|------|------|
| P30 | 复制 toast 文案不一致 | renderer.js 各 copy 处理 | `copyPwd` 显示"已复制到剪贴板"、`copyPhrase` 也是、批量小复制显示"已复制"、"复制全部"显示"已复制全部"——4 种不同文案，且不带数量信息 |
| P31 | 批量"复制全部"无数量反馈 | renderer.js `batchCopyAll` | toast 仅"已复制全部"，未告知具体复制了几个密码 |
| P32 | 清空历史的 armed 状态在切换 Tab 后仍保留 | renderer.js tab click | 用户点击"清空历史"（按钮变红显示"再次点击确认清空"），切到其他 Tab 再切回，按钮仍为 armed 态但用户已忘记，可能误点确认 |
| P33 | 底部状态栏与上方内容视觉脱节（mimo） | styles.css `.statusbar` | 状态栏仅有 1px 浅边框，与上方卡片之间缺少明确分隔，mimo 指出"fused to page, no separation" |
| P34 | 强度输入框无 maxlength | index.html `strengthInput` | 用户可粘贴超长文本（MB 级），潜在性能风险 |

## 三、修复方案（逐条对应）

### 修复 P21 — 复制按钮短暂 ✓ 视觉反馈
新增 `flashCopyFeedback(btn)` 工具函数：点击复制后按钮临时变绿色 + 图标替换为 ✓，900ms 后恢复原 SVG。
- 应用于：`copyPwd` / `copyPhrase` 两个图标按钮
- 批量项 / 历史项的小"复制"文本按钮：原文本临时替换为 `✓` + `.copied` 类变绿背景，800ms 后恢复

### 修复 P22 — 滑块拖动期间防抖刷新密码
新增 `sliderPreviewTimer` + `debouncedGenPassword()`：滑块 `input` 事件触发 150ms 节流刷新 `genPassword(true)`。
- 用户拖动时数字输入框、滑块填充色立即更新
- 拖动停顿 150ms 后密码展示区、强度条、元信息全部刷新到新长度
- 拖动结束 `change` 事件仍兜底再生成一次

### 修复 P23 — 字符类型 / 分隔符改变自动重新生成
- 5 个字符类型 checkbox（`chkUpper/chkLower/chkDigits/chkSymbols/chkExcludeAmb`）绑 `change` 事件 → `genPassword(true)`
- 4 个分隔符 segmented 按钮 click 后立即 `genPassphrase(true)`
- 2 个口令选项 checkbox（`chkCapitalize/chkNumber`）绑 `change` → `genPassphrase(true)`
- 与滑块 `change` 行为对齐，所有配置改动后展示区即时更新

### 修复 P24 — Ctrl+G 在不可生成 Tab 给反馈
keydown 处理增加 `else` 分支：在 strength / history Tab 按 Ctrl+G 时 `showToast('当前页不支持 Ctrl+G 生成')`，避免静默无效。
- 同时扩展 Ctrl+Shift+C：在 batch Tab 且已有批量结果时直接复制全部，否则提示"当前页无内容可复制"

### 修复 P25 — 复制按钮 disabled 视觉态
- 新增 `syncCopyDisabled()` 函数：密码/口令展示为空（"点击生成开始"）时给 `copyPwd` / `copyPhrase` 设 `disabled=true`
- 在 `genPassword` / `genPassphrase` 末尾、初始化时调用
- CSS 新增 `.icon-btn:disabled` 样式：opacity 0.4、cursor not-allowed、pointer-events none，灰化禁用态

### 修复 P26 — 弱密码配置警告
新增 `weakConfigWarning(opts)` 函数，`genPassword` 非静默模式下生成前检查：
- 长度 < 8：toast `当前长度<8，密码强度极弱，仅用于PIN等场景`
- 仅数字 + 长度 < 10：toast `纯数字+短长度易被暴力破解，建议加字母/符号或增加长度`
- 用户使用 PIN 预设（4 位纯数字）时会看到警告，知情使用

### 修复 P27 — 长度 / 单词数量输入框 Enter 确认
- `lengthInput` keydown Enter → `genPassword()`（非静默，写入历史 + 状态栏更新）
- `wordInput` keydown Enter → `genPassphrase()`（同上）
- 用户输入数字后直接 Enter 即生成，无需鼠标点击

### 修复 P28 — 移除强度页 label 与 placeholder 的文案重复
HTML label 从 `输入待检测密码` 改为 `待检测密码`（去掉"输入"两字），placeholder 保留 `在此输入或粘贴密码…`。
- 同时为输入框加 `maxlength="256"`（修复 P34）

### 修复 P29 — 强度条 bits 标签对齐
`.strength-info` 加 `align-self: stretch` 让其与强度条左右两端对齐；`.strength-info span:first-child` 与 `.strength-hint` 改为 `display: inline-flex; align-items: center`，避免文字基线飘空。

### 修复 P30 — 统一复制 toast 文案 + 带数量
- 密码：`已复制 16 位`
- 口令：`已复制 21 字符`
- 批量单项：`已复制 16 位`
- 批量全部：`已复制全部 10 个`
- 空内容：统一为 `暂无内容可复制`（去掉"暂无内容"）

### 修复 P31 — 批量"复制全部"带数量反馈
`batchCopyAll` 的 toast 从 `已复制全部` 改为 `已复制全部 N 个`，用户知道具体复制了几个。

### 修复 P32 — 切换 Tab 复位清空历史 armed 态
新增 `resetClearHistoryArmed()` 函数，在 Tab click 事件中调用：
- 若 armed 态为 true，立即复位：清 timer、还原按钮文字为"清空历史"、移除 `.armed` 类
- 避免用户跨 Tab 来回时忘记 armed 状态而误点确认

### 修复 P33 — 状态栏视觉分隔加强
`.statusbar`：
- 顶边从 `1px solid var(--border)`（极浅）升级为 `1px solid var(--border-strong)`（实色）
- 加 `box-shadow: 0 -1px 0 rgba(0,0,0,0.02)` 微妙分隔感
- 背景透明度 0.85 → 0.92，padding 8px → 9px
- 状态栏与上方内容有明确视觉断开

### 修复 P34 — 强度输入框 maxlength
`strengthInput` 加 `maxlength="256"`，防止用户粘贴超长文本导致的性能问题。

## 六、结论

本轮在上一轮 20 个修复的基础上，又发现并修复 **14 个** 更细节的 UX 问题（🔴 严重 3 / 🟡 中等 6 / 🟢 轻微 5），全部当场修复并重新验证。

mimo 审美评分从 **8.0 → 9.0**（+1.0），其中强度检测页从 7.5 → **9.2**（+1.7）提升最显著，主要得益于状态栏分隔加强、强度条对齐修复、卡片视觉统一。

核心交互改善：
- **复制反馈闭环**：从无反馈到 ✓ 闪现 + 数量 toast，频繁操作有即时确认
- **配置即所见**：拖动滑块 / 切换字符类型 / 切换分隔符后展示区即时刷新，消除"改了配置但展示不变"的认知错位
- **快捷键完整闭环**：Ctrl+G 在所有 Tab 都有反馈，Ctrl+Shift+C 支持批量页
- **弱配置知情同意**：PIN / 短数字密码生成时主动提示风险
- **禁用态明确**：空内容时复制按钮灰化，避免无效点击

修复细节见同次提交的源码 diff。
