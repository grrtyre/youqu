## 三、代码审计发现的问题

### 🔴 严重（Critical）

**F1. `launch-app` 启动失败用户无感知**
- 位置：[src/main.js#L215-L226](src/main.js)
- 现象：`shell.openPath(appPath)` 是异步的，但代码用 try/catch 包裹后立即返回 `{ success: true }`。实际启动失败（路径无效、权限不足）时，用户只看到窗口隐藏，毫无反馈
- 修复：await `shell.openPath` 返回值（错误字符串或空字符串），失败时返回错误并显示提示

**F2. `shortenPath` 只覆盖两种路径模式**
- 位置：[src/renderer/renderer.js#L200-L202](src/renderer/renderer.js)
- 现象：只处理 `C:\Users\<name>\` 和 `C:\ProgramData\`，未处理 `C:\Program Files\`、`C:\Users\Public\`、`D:\`、`E:\` 等常见路径，导致路径显示又长又丑
- 修复：补充更多常见路径前缀的缩短规则

**F3. 无系统托盘，窗口关闭后无法唤起**
- 位置：[src/main.js#L180-L182](src/main.js)
- 现象：`window-all-closed` 阻止退出，但应用没有系统托盘图标。如果用户忘记 Alt+Space 快捷键，会以为应用"消失"了
- 修复：添加 `Tray` 图标 + 右键菜单（显示/退出），与便携版 PySide6 行为一致

### 🟡 中等（Medium）

**V1. 底部状态栏与主内容割裂**（mimo 指出）
- 现象：1px 细线硬分隔，状态栏背景 `#f5f5f7` 与主区 `#ffffff` 反差大，像两个组件拼接
- 修复：用极浅渐变 + 取消硬分割线 + 状态栏背景与主区同色，仅靠留白区分

**V2. 状态栏快捷键提示拥挤**（mimo 指出）
- 现象："↑↓ 选择 · ↵ 启动" 间距 14px，中文与符号混排拥挤
- 修复：扩大间距至 18px，补 Esc 提示，统一字符样式

**V3. 单条结果时弹窗空旷**（mimo 指出）
- 现象：窗口固定 460px 高度，结果少时下方大量留白
- 修复：窗口高度根据结果数量动态调整（最少 5 项高度，最多保持原值）

**V4. 顶部 hint-kbd "Alt+Space" 与输入框争夺注意力**（mimo 指出）
- 现象：右上角蓝灰色 hint-kbd 视觉权重偏高，与输入框光标焦点冲突
- 修复：弱化 hint-kbd 样式（更浅色、更小、移到状态栏统一展示）

**V5. 空状态文案不区分场景**
- 现象：无查询时和有查询无结果都显示"未找到匹配的应用"，但无查询时根本没搜索
- 修复：无查询且无索引时显示"正在索引应用..."；有查询无结果显示"未找到匹配的应用"

**V6. 搜索防抖 80ms 偏长**
- 位置：[src/renderer/renderer.js#L67-L69](src/renderer/renderer.js)
- 现象：80ms 防抖让快速打字有延迟感（Spotlight/Alfred 通常 30-50ms）
- 修复：降至 40ms

**V7. 鼠标 hover 触发全量 re-render**
- 位置：[src/renderer/renderer.js#L151-L154](src/renderer/renderer.js)
- 现象：`mouseenter` 调用 `renderResults()` 重建整个 DOM，8 项时也明显闪烁
- 修复：只更新 `.selected` 类，不重建 DOM

**V8. 路径无 tooltip**
- 现象：路径用 ellipsis 截断，鼠标悬停看不到完整路径
- 修复：`item.title = app.path` 显示原生 tooltip

### 🟢 轻微（Minor）

**D1. `isUnwanted` 过滤"help"过于宽泛**
- 位置：[src/lib/appIndexer.js#L76-L80](src/lib/appIndexer.js)
- 现象：会过滤 "HelpDesk"、"HelpScout"、"Eclipse Help" 等正常应用
- 修复：用词边界匹配（仅匹配独立的 "Help" / "帮助"）

**D2. 首次启动无加载提示**
- 现象：indexer 后台扫描时，状态栏直接显示"0 个应用已索引"且无 spinner
- 修复：扫描中显示"正在索引..."，扫描完成显示具体数量

**D3. `setInterval` 每 5 分钟全量重扫**
- 位置：[src/main.js#L161-L165](src/main.js)
- 现象：性能浪费，应该用更长的间隔或文件系统监听
- 修复：间隔延长至 15 分钟（暂不引入 fs.watch 复杂度）

**D4. `autofocus` 属性 + 手动 focus() 冗余**
- 位置：[src/renderer/index.html#L23](src/renderer/index.html) + [renderer.js#L51](src/renderer/renderer.js)
- 现象：HTML 的 `autofocus` 与 JS 的 `searchInput.focus()` 都会触发焦点，可能引起双重事件
- 修复：移除 HTML 的 `autofocus`，统一由 JS 控制

## 四、修复实施

### F1. launch-app 真实反馈
```js
ipcMain.handle('launch-app', async (event, appPath) => {
  try {
    saveRecent(appPath);
    const err = await shell.openPath(appPath);
    if (err) {
      return { success: false, error: err };
    }
    if (mainWindow && mainWindow.isVisible()) {
      mainWindow.hide();
    }
    return { success: true };
  } catch (e) {
    return { success: false, error: e.message };
  }
});
```

### F2. shortenPath 扩展
```js
function shortenPath(p) {
  return p
    .replace(/^[A-Z]:\\Users\\[^\\]+\\/, '~/')
    .replace(/^[A-Z]:\\ProgramData\\/, '/ProgramData/')
    .replace(/^[A-Z]:\\Program Files( \(x86\))?\\/, '/Program Files/')
    .replace(/^[A-Z]:\\Users\\Public\\/, '/Public/')
    .replace(/^[A-Z]:\\Windows\\/, '/Windows/');
}
```

### F3. 系统托盘
新增 `Tray` + `Menu.buildFromTemplate`，托盘点击唤起，右键菜单含"显示启动器/退出"。

### V1-V8 视觉与交互修复
- 状态栏去分割线 + 同色背景 + 扩大间距
- 窗口高度根据结果数量动态调整
- hint-kbd 弱化
- 空状态文案区分
- 防抖 40ms
- hover 仅切换 selected 类
- path 添加 title 属性

### D1-D4 细节修复
- isUnwanted 词边界匹配
- 索引中显示"正在索引..."
- 扫描间隔 15 分钟
- 移除 HTML autofocus
