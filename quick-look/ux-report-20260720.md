最严重的问题是 **blur 自动隐藏 bug**：`ready-to-show` 后立即触发 `blur` 事件，120ms 延迟不够，导致窗口在首次启动后立刻消失，用户根本看不到欢迎页。这是阻断式故障，必须最高优先级修复。

## 一、🔴 严重问题（7 个）

### S1. blur 自动隐藏导致首次启动窗口消失
- **位置**：`src/main.js:111-121`
- **现象**：`ready-to-show` 后调用 `showWindow()`，`show()` + `focus()` 过程中窗口瞬间失焦，触发 `blur` → 120ms 后调用 `hideWindow()`，欢迎页一闪而过。
- **根因**：首次启动没有 grace 宽限期；`setTimeout 120ms` 太短，无法覆盖 Electron 启动期间的焦点抖动。
- **修复**：引入 `startupGraceUntil` 时间戳（启动后 1.5 秒内忽略 blur 隐藏）；并增加 `suppressBlurHide` 标志位供其他流程临时禁用。

### S2. 拖拽文件过程中 blur 误隐藏
- **位置**：`src/main.js` + `src/renderer/renderer.js`
- **现象**：用户从资源管理器拖文件到 dropzone 时，鼠标移动过程可能短暂离开窗口区域，触发 blur，窗口在拖拽途中消失。
- **修复**：渲染进程通过 IPC 通知主进程 `drag-start` / `drag-end`，期间 `suppressBlurHide = true`。

### S3. 点击 ghost-btn 后窗口被外部应用抢占导致隐藏
- **位置**：`src/main.js` + `src/renderer/renderer.js`
- **现象**：用户点击"系统打开""显示位置"按钮后，系统会启动外部应用抢焦点，回来时窗口已被 blur 隐藏。
- **修复**：ghost-btn 点击后 `suppressBlurHide = true`，1.5 秒后自动恢复。

### S4. 托盘菜单"打开文件选择"对话框期间 blur 隐藏
- **位置**：`src/main.js:167-178`
- **现象**：点击托盘"打开文件选择"→ `showWindow()` → `dialog.showOpenDialog()`，对话框期间主窗口失焦被隐藏，用户选完文件回来窗口没了。
- **修复**：托盘菜单点击后 `suppressBlurHide = true`，dialog 关闭后 1 秒恢复。

### S5. 窗口被遮挡时 Alt+Q 不置顶
- **位置**：`src/main.js:128-135` `showWindow()`
- **现象**：窗口 visible 但被其他窗口遮挡，按 Alt+Q 调用 `focus()` 但不会把它抬到最顶层，用户以为快捷键失效。
- **修复**：`showWindow()` 中先 `setAlwaysOnTop(true)` → `focus()` → 短暂延迟后 `setAlwaysOnTop(false)`，模仿 macOS QuickLook 行为。

### S6. 托盘图标可能为空
- **位置**：`src/main.js:154-162`
- **现象**：如果 `build/icon.ico` 不存在或加载失败，`tray = new Tray(nativeImage.createEmpty())`，Windows 任务栏会出现一个空白图标，用户找不到应用。
- **修复**：用 `nativeImage.createFromBuffer` 生成一个 16×16 蓝色圆点 PNG 作为默认托盘图标，保证始终可见。

### S7. 快捷键注册失败无用户提示
- **位置**：`src/main.js:210-212`
- **现象**：`globalShortcut.register` 失败时只 `console.error`，用户毫不知情，按 Alt+Q 无反应以为应用坏了。
- **修复**：注册失败时通过 `dialog.showMessageBox` 弹出提示告知用户，并更新托盘 tooltip。

## 二、🟡 中等问题（8 个）

### M1. 首次启动无引导
- **位置**：`src/renderer/renderer.js` init()
- **现象**：第一次启动直接显示欢迎页，用户看完关掉后不知道怎么再唤起。
- **修复**：检测 `localStorage.firstRun`，首次启动 800ms 后显示 Toast「按 Alt+Q 随时唤起我」。

### M2. 空文件预览无反馈
- **位置**：`src/renderer/renderer.js` renderText
- **现象**：加载 0 字节文本文件时显示空白 `<pre>`，用户以为是 bug。
- **修复**：检测 `content === ''`，显示「文件为空」占位卡片。

### M3. 超长文件名导致 title-text 溢出
- **位置**：`src/renderer/styles.css` + `renderer.js`
- **现象**：文件名超长时 `.title-text` 居中显示会盖住右侧按钮，或被截断不可读。
- **修复**：`.title-text` 加 `max-width: 50vw; overflow: hidden; text-overflow: ellipsis; white-space: nowrap;`。

### M4. 历史路径过长水平溢出
- **位置**：`src/renderer/styles.css:465-472`
- **现象**：`.h-path` 没有 `word-break`，长路径撑开抽屉宽度或被截断不可读。
- **修复**：加 `word-break: break-all; white-space: normal;`。

### M5. 清空历史无二次确认
- **位置**：`src/renderer/renderer.js:105-109`
- **现象**：点击"清空"立即清掉全部历史，误操作无法恢复。
- **修复**：第一次点击切换为"确认清空？"红色按钮，3 秒内再点才真正清空，否则恢复。

### M6. Markdown 链接无法点击
- **位置**：`src/renderer/renderer.js:355` simpleMarkdown
- **现象**：`<a href="$2">` 在 CSP `default-src 'self'` 下点击无效。
- **修复**：渲染后给所有 `a` 拦截 click，调用 `window.api.shell.open`。

### M7. 视频/音频路径含特殊字符无法加载
- **位置**：`src/renderer/renderer.js:239, 247, 264, 299`
- **现象**：`file://${path.replace(/\\/g, '/')}` 没有转义 `#`、`?`、空格，含这些字符的路径加载失败。
- **修复**：用 `encodeURI` 转义路径。

### M8. 窗口启动时尺寸可能超出屏幕
- **位置**：`src/main.js:82-101`
- **现象**：配置中 `windowWidth/Height` 可能保存了旧分辨率下的值（如 2560×1440），换到 1366×768 屏幕后窗口超出屏幕。
- **修复**：启动时读取 `screen.getPrimaryDisplay().workAreaSize`，超过则限制到 90%。

## 三、🟢 轻微问题（5 个）

### L1. dropzone hover 动画 nth-child 限制
- **位置**：`src/renderer/styles.css:254-258`
- **现象**：硬编码 5 个 `nth-child`，扩展图标数量会漏。
- **修复**：改为 `.dropzone:hover .ft-icon` 通用规则 + `transition-delay: calc(var(--i) * 50ms)`。

### L2. 滚动条 thumb 对比度不足
- **位置**：`src/renderer/styles.css:279`
- **现象**：`#d0d0d4` 在浅背景下几乎看不见。
- **修复**：改为 `#b8b8bc`，hover 时 `#9a9a9f`。

### L3. Toast 位置可能被 meta-bar 遮挡
- **位置**：`src/renderer/styles.css:482`
- **现象**：`bottom: 60px` 在预览模式下与 meta-bar 重叠。
- **修复**：改为 `bottom: 80px`。

### L4. 历史抽屉切换无过渡动画
- **位置**：`src/renderer/styles.css:417-429`
- **现象**：`hidden` 切换瞬间出现/消失，突兀。
- **修复**：改为 `transform: translateX(100%)` + `transition: transform 0.25s ease`，用 `.open` 类控制。

### L5. unknown 文件用错误样式显示
- **位置**：`src/renderer/renderer.js` showError
- **现象**：unknown 分类走 `showError('暂不支持此文件格式')`，但视觉上是「错误」红色警告样式，给用户挫败感。
- **修复**：新增 `showUnsupported(meta)` 用中性灰色 info 样式，展示文件元信息 + 「点击下方系统打开」提示。
