## 二、本轮发现并修复的问题

### 🔴 严重（崩溃级，HTML/JS 不同步导致）

**P1. 自定义确认弹窗 HTML 元素完全缺失**
- 现象：`renderer.js` 引用 `confirmOverlay` / `confirmIcon` / `confirmTitle` / `confirmMessage` / `confirmCancelBtn` / `confirmOkBtn` 6 个 DOM 元素，但 `index.html` 中完全没有这些元素（历史迭代只改了 JS 和 CSS，漏提交 HTML）
- 影响：任何调用 `showConfirm()` 的操作立即 TypeError 崩溃，包括：清空回收站、彻底删除单条、关闭未保存弹窗。**用户碰一次删除就崩**
- 修复：在 `index.html` 末尾添加完整的 `.confirm-overlay > .confirm-dialog` 结构，包含图标容器、标题、消息、取消/确认按钮，加 `role="dialog" aria-modal="true" aria-labelledby` 无障碍属性

**P2. 搜索清除按钮 `searchClearBtn` 元素缺失**
- 现象：`renderer.js` 第 77 行 `getElementById('searchClearBtn')` 返回 null，第 148/159/647/655 行访问 `.style.display` 立即崩溃
- 影响：进入回收站视图（`updateViewControls` 触发）即崩溃；在搜索框输入文字即崩溃
- 修复：在 `.search-box` 内添加完整的清除按钮 SVG（带 `aria-label="清除搜索"`），默认 `display:none`

**P3. 空状态图标容器 `emptyIcon` id 缺失**
- 现象：`renderer.js` 在 `renderNotes()` / `renderTrash()` 中通过 `getElementById('emptyIcon')` 切换不同空状态图标，但 HTML 中 `.empty-icon` 没有 id
- 影响：切换「无搜索结果」「回收站为空」「无便签」三种空状态时图标永远不更新，且控制台报错
- 修复：在 `.empty-icon` div 上加 `id="emptyIcon"`

### 🟡 中等（UX 改进）

**P4. 回收站视图下「新建便签」按钮被禁用**
- 现象：上轮代码 `newNoteBtn.style.opacity='0.5'; pointerEvents='none'`，用户在回收站想新建便签必须先切回分类
- 修复：回收站视图也允许点击新建按钮，点击时自动切回便签视图并打开新建弹窗；调整 `newNoteBtn` 的 `title` 提示行为

**P5. 排序方式无交互，仅显示文字**
- 现象：`sortInfo` 是纯文本 `<div>`，用户无法切换排序方式（mimo 第二轮反馈「顶部排序文字与标题关联弱，可改为下拉选择器」）
- 修复：改为 `<select id="sortSelect">`，支持 4 种排序：置顶优先·按更新时间 / 按更新时间 / 按创建时间 / 按标题字母序；自定义苹果白风格外观 + 自定义箭头图标

**P6. 删除便签无「撤销」操作**
- 现象：上轮报告未修复项。删除后只能去回收站手动恢复，违反 Apple HIG「破坏性操作应提供 5 秒撤销窗口」
- 修复：`deleteNote()` 触发后展示带「撤销」按钮的 Toast（5 秒持续），点击撤销直接把便签从 trash 移回 notes 并保留原 id/字段；`showToast()` 函数升级支持可选 `undo` + `undoFn` 参数

**P7. 便签网格无键盘方向键导航**
- 现象：上轮报告未修复项。卡片虽有 `tabindex="0"` 可 Tab 聚焦，但方向键无法在卡片间移动
- 修复：在 `notesGrid` 上绑定 keydown，根据当前卡片最小宽度（248px）+ gap（24px）动态估算每行卡片数，支持 ArrowUp/Down/Left/Right 移动焦点；弹窗打开时不响应

**P8. 编辑弹窗输入框无 maxlength 限制**
- 现象：超长标题/正文会撑破卡片布局
- 修复：标题 `maxlength="120"`，正文 `maxlength="20000"`

**P9. tray menu accelerator 与全局快捷键不一致**
- 现象：tray menu「新建便签」`accelerator: 'CmdOrCtrl+N'`，但实际全局快捷键是 `Ctrl+Alt+N`；tray menu「显示主窗口」无 accelerator
- 修复：分别改为 `CommandOrControl+Alt+N` / `CommandOrControl+Alt+S`，与全局快捷键一致

**P10. main.js `app.dock` 重复条件 + Windows 无 dock API**
- 现象：`app.dock && app.dock && app.dock.hide()` 重复 `app.dock` 判断，且 Windows 上 `app.dock` 为 undefined，未做类型检查
- 修复：简化为 `if (app.dock && typeof app.dock.hide === 'function') app.dock.hide()`

**P11. `createNote` 强制覆盖 updatedAt 导致导入数据丢失原始时间戳**
- 现象：`note-store.js` 中 `createNote()` 第 46 行 `updatedAt: now` 硬编码覆盖；`importNotes()` 调用 `createNote()` 后所有导入便签 `updatedAt` 变为当前时间，导致演示数据导入后全部显示「刚刚」（上轮报告遗留问题）
- 修复：`createNote()` 改为 `updatedAt: typeof data.updatedAt === 'number' && data.updatedAt > 0 ? data.updatedAt : now`；`importNotes()` 显式透传 `createdAt` / `updatedAt`

**P12. 导入失败 Toast 文案过于技术性**
- 现象：`showToast('导入失败：' + result.error)` 直接暴露 `Unexpected token...` 等 JSON 解析错误
- 修复：根据 error 文本匹配常见错误类型，给出友好文案（如「文件格式不正确，请选择便签管家导出的 JSON 文件」）；技术细节打印到 `console.error`

**P13. 导出 Toast 路径过长撑破布局**
- 现象：`showToast('已导出到 ' + result.path)` 路径可能很长
- 修复：截断只显示文件名（按 `\` 或 `/` 分割取最后一段）；导出取消时也提示「已取消导出」；空便签时阻止导出并提示

**P14. 颜色圆点默认 border 在白底上不可见**
- 现象：`.color-opt { border: 2px solid #fff }` 在白色背景上完全不可见，未选中态圆点缺少边界
- 修复：改为 `border: 2px solid var(--bg-primary)` + `box-shadow: 0 0 0 1px var(--border)` 让圆点在白底上有清晰边界

### 🟢 轻微（视觉/微交互细节）

**P15. 侧边栏底部缺少版本号显示**
- 修复：在 footer-actions 下方加 `<div class="sidebar-version">便签管家 v1.1.0</div>`

**P16. 卡片左侧色条与分类 pill 视觉冲突**
- 现象：mimo 第二轮反馈。border-left 3px 色条 + 底部分类 pill 同时传达颜色信息，视觉冗余
- 修复：色条减为 2px，阴影改为 Apple 风柔和漫射（`0 1px 2px rgba(0,0,0,0.04), 0 2px 6px rgba(0,0,0,0.04)`）

**P17. 统计区容器背景与列表风格断裂**
- 现象：mimo 反馈。`.stats-card` 用了 `bg-secondary` 容器，与上方无背景的 `.category-item` 风格不一致
- 修复：移除背景，改为透明轻量布局

**P18. 删除按钮实心红色过强**
- 现象：mimo 反馈。`.btn-delete` 浅红背景 + 红边框视觉权重过强
- 修复：改为幽灵样式（透明背景 + 红字 + 细红边框），hover 时才出现浅红背景

**P19. 新建按钮视觉权重过大**
- 现象：mimo 反馈。padding 11px、font-size 13px、box-shadow 0 2px 8px 视觉过重
- 修复：减为 padding 9px、font-size 12.5px、box-shadow 0 1px 3px 更柔和

**P20. 卡片网格间距偏窄**
- 现象：mimo 反馈。gap: 20px + grid-auto-rows 184px 在 1200px 宽度下显得拥挤
- 修复：gap 改为 24px，grid-auto-rows 改为 188px

**P21. 按钮无障碍属性补全**
- 修复：modal-close 加 `title="关闭 (Esc)"`，deleteBtn 加 `title="移入回收站（可恢复）"`，saveBtn 加 `title="保存 (Ctrl+Enter)"`，charCount 加 `title` 说明字数统计规则；所有 meta-opt/color-opt 加 `type="button"` 防止表单提交；color-opt 加 `aria-label` 描述颜色名

## 六、遗留问题与下轮迭代建议

1. **mimo 评分稳定性**：mimo 对同一张截图会反复指出"阴影不够"、"圆角不够大"等主观偏好，与苹果白克制美学存在内在冲突。建议保持当前方向，不盲目追高分数
2. **PrintWindow 截图稳定性**：本轮改用 Edge headless + 自包含测试 HTML，比 PrintWindow 稳定得多，可作为后续巡检的标准方案
3. **多语言文案**：所有文案目前为中文硬编码，可考虑后续 i18n
4. **主题切换**：当前仅苹果白主题，可考虑加深色主题作为用户选项（默认仍为苹果白）
