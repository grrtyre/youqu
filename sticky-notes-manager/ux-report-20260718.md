## 二、本轮发现并修复的问题

### 🔴 严重（视觉一致性破坏）

**P1. 原生 confirm 弹窗破坏苹果白风格**（第一轮已修，本轮强化）
- 现象：原生 confirm 与应用整体设计语言冲突
- 修复：自定义 Promise 化确认弹窗，加入图标背景色（danger 红 / info 蓝）、危险按钮独立配色、阴影增强、按钮 hover 上移动画

**P2. 窗口位置/尺寸不持久化**（第一轮已修）
- 现象：每次启动窗口回到默认位置
- 修复：window-state.json 持久化 x/y/width/height/isMaximized，500ms 防抖保存

### 🟡 中等（视觉细节）

**P3. 圆角不统一**（新发现）
- 现象：sidebar/button 8px、card 14px、modal 16px，三种圆角混杂
- 修复：统一为 8px 网格 → sm=10px / md=12px / lg=14px，差距收窄至 2px

**P4. 阴影层次感不足**（新发现）
- 现象：sm/md/lg 三种阴影差异不明显，缺乏深度
- 修复：阴影改为双阴影叠加（主阴影 + 微小近距阴影），层次更分明

**P5. 卡片左侧色条过粗**（mimo 反复指出）
- 现象：border-left: 5px 视觉重量过大，扫视时被边框吸引
- 修复：减为 3px，padding-left 22px 补偿留白

**P6. 毛玻璃过度模糊**（mimo 指出）
- 现象：modal-overlay backdrop-filter: blur(4px) 背景几乎不可见
- 修复：减为 blur(2px)，背景遮罩 0.25 → 0.28 微调对比

**P7. 编辑弹窗关闭按钮视觉权重低**（mimo 指出）
- 现象：32px 圆形按钮颜色过浅易忽略
- 修复：36px + flex-shrink:0 + hover 时背景加深至 var(--border) + scale(1.05) 微动效

**P8. 删除按钮纯文字链接样式不足**（mimo 指出）
- 现象：btn-delete 是 transparent 背景 + 红色文字，危险操作视觉警示不足
- 修复：改为 #fff5f5 浅红背景 + 1px #ffd6d6 红边框，hover 时颜色加深 + translateY(-1px)

**P9. 颜色选择器与分类标签间距过小**（mimo 指出）
- 现象：meta-options gap:6px 易误触
- 修复：gap 改为 8px，分类标签加边框 + hover 状态 + 选中态阴影

**P10. 侧边栏与主内容区无分隔**（mimo 指出）
- 现象：border-right 用 --border-light（过浅），层次弱
- 修复：改用 --border（更深）+ 1px box-shadow 双重分隔

**P11. 回收站卡片边框过浅**（mimo 指出）
- 现象：trash-row border-light 与背景融合度过高
- 修复：改为 --border + box-shadow var(--shadow-sm) + hover translateY(-1px) 增强交互反馈

**P12. 回收站间距过大**（mimo 指出）
- 现象：trash-list gap:12px 显得松散
- 修复：gap 改为 10px，trash-row border-left 4px → 3px（与便签卡片一致）

**P13. 分类项间距过紧**（mimo 指出）
- 现象：category-item margin-bottom:2px 与 22px 卡片间距比例失调
- 修复：margin-bottom 改为 4px，sidebar 宽度 280px → 288px 提升呼吸感

**P14. 确认弹窗按钮层级弱**（mimo 指出）
- 现象：主操作与取消按钮尺寸/字重接近，主操作未形成视觉焦点
- 修复：padding 9px → 11px（更大触摸区）+ 主操作加 0 2px 8px 阴影 + hover 上移 + 加深阴影

**P15. 确认弹窗图标背景缺失**（新发现）
- 现象：图标无背景容器，与 macOS 风格弹窗不一致
- 修复：图标外层加 56x56 圆形容器 + danger 时红底 rgba(255,59,48,0.08) + info 时蓝底 rgba(0,122,255,0.08)

**P16. 卡片网格尺寸不规范**（新发现）
- 现象：minmax(250px) + 180px 固定高度 + 22px gap 不符合 8px 网格
- 修复：minmax(248px) + 184px 高度 + 20px gap，对齐 8px 网格系统

### 🟢 轻微（微交互）

**P17. 分类标签 hover 反馈缺失**（新发现）
- 修复：meta-opt hover 时背景 var(--bg-primary) + 边框 var(--border) + 文字加深

**P18. 卡片 hover 阴影过淡**（mimo 指出）
- 修复：trash-row hover 加 var(--shadow-md) + translateY(-1px)

**P19. 确认弹窗标题字重不足**（mimo 指出）
- 修复：confirm-title 16px → 17px + letter-spacing -0.3px，视觉层次加强

**P20. 模态背景对比不足**（mimo 指出）
- 修复：modal-overlay 0.25 → 0.28，confirm-overlay 同步

## 六、遗留问题与下轮迭代建议

1. **PrintWindow 截图稳定性**：本轮第二轮截图失败，怀疑 Windows DWM 缓存问题。下轮迭代可改用 Edge headless 渲染独立 HTML 测试页（需 mock preload.js 的 IPC 接口）作为补充方案
2. **mimo 评分未达 9/10 目标**：本次评分最高 7.5/10。mimo 评分特性是会反复指出"圆角不够大"、"阴影不够明显"等主观偏好，需在保持苹果白克制美学的前提下选择性采纳
3. **演示数据时间戳**：ux-demo-data.js 应在 createNote 时设置不同的 updatedAt，避免全部显示"刚刚"
4. **顶部排序控件**：可考虑改为下拉选择器或移至标题下方
5. **统计区**：sidebar-footer 可加分隔线 + padding-top 强化层次
