## 一、巡检范围

通过 GitHub API 获取 grrtyre/youqu 仓库 sticky-notes-manager 源码，对以下文件做完整 UX 审查：
- `src/main.js`（主进程：窗口、托盘、全局快捷键、IPC）

- `src/core/note-store.js`（核心存储：CRUD、回收站、搜索、排序、统计）

模拟的用户流程：新建便签 → 编辑（标题/正文/分类/颜色）→ 保存 → 右键置顶 → 搜索 → 分类筛选 → 删除（移入回收站）→ 回收站恢复/彻底删除/清空 → 导入导出 → 快捷键（Ctrl+Enter、Esc、Ctrl+Alt+N/S）→ 关闭弹窗/最小化到托盘。

## 二、发现的问题与修复

### 🔴 严重

#### 1. 清空回收站无二次确认，误点即永久丢失全部
- **位置**：`renderer.js` `emptyTrash()`
- **问题**：点击「清空回收站」直接执行，无确认弹窗。这是不可逆破坏性操作，违反 Apple HIG「对不可逆操作必须二次确认」原则。
- **修复**：增加 `confirm()` 确认弹窗，显示将删除的条数并明确「不可恢复」。

#### 2. 回收站「彻底删除」单条无二次确认
- **位置**：`renderer.js` `deleteFromTrash(id)`
- **问题**：单条彻底删除同样无确认，一点即永久删除。
- **修复**：增加 `confirm()`，携带便签标题提示「不可恢复，删除后无法找回」。

#### 3. 编辑弹窗关闭时未保存内容直接丢失
- **位置**：`renderer.js` `closeModal()`、遮罩点击、Esc 关闭
- **问题**：用户输入了大量内容后，误点遮罩或按 Esc，弹窗直接关闭，内容全部丢失无任何提示。数据丢失风险高。
- **修复**：新增 `hasUnsavedChanges()` 检测函数，对比当前输入与原便签（标题/正文/分类/颜色）；存在未保存内容时弹确认。保存/删除流程使用 `closeModal(true)` 强制关闭避免误触确认。

### 🟡 中等

#### 4. Toast 文案错误：更新便签时提示「已创建」
- **位置**：`renderer.js` `saveNote()` 末尾
- **问题**：`closeModal()` 先于 `showToast()` 执行，导致 `editingNoteId` 被置空，`editingNoteId ? '已保存' : '已创建'` 恒为「已创建」。更新已有便签也提示「已创建」，误导用户。
- **修复**：在 `closeModal()` 前用 `wasUpdating` 缓存状态，Toast 据此正确显示。

#### 5. 进入回收站后搜索状态不一致
- **位置**：`renderer.js` `updateViewControls()`
- **问题**：进入回收站时 `searchInput.value` 被清空，但 `currentSearch` 未同步清空。返回便签视图后搜索框显示为空，但列表仍被旧关键词过滤，视觉与状态错位，用户困惑。
- **修复**：进入回收站时同步 `currentSearch = ''`。

#### 6. 回收站剩余天数颜色无分级，临近过期不警告
- **位置**：`renderer.js` `renderTrash()` + `styles.css`
- **问题**：剩余天数恒为橙色 `#ff9500`，无论还剩 1 天还是 25 天都一样，临近过期缺乏视觉紧迫感。
- **修复**：按剩余天数分级——≤1 天红色加粗（`trash-days-urgent`）、≤3 天橙色（`trash-days-warn`）、其余灰色（`trash-days`）。

#### 7. 标题输入框不支持 Ctrl+Enter 保存
- **位置**：`renderer.js` keydown 绑定
- **问题**：仅正文 textarea 绑定 Ctrl+Enter 保存，标题 input 无此快捷键。用户在标题输入完按 Ctrl+Enter 无反应，快捷键不一致。
- **修复**：抽取 `ctrlEnterSave` 函数，标题与正文均绑定。

#### 8. 回收站「删除」按钮纯图标，与「恢复」按钮风格不统一
- **位置**：`renderer.js` `renderTrash()` + `styles.css`
- **问题**：「恢复」按钮是图标+文字，「删除」按钮仅图标，两者并列时辨识度低、风格不统一。
- **修复**：「删除」按钮增加「删除」文字，样式调整为图标+文字+边框，与「恢复」按钮风格对齐；补充 `aria-label`。

#### 9. README 版本号与测试用例数前后矛盾
- **位置**：`README.md`
- **问题**：安装说明写「便签管家 Setup 1.0.0.exe」但实际版本 1.1.0；测试章节写「86 个用例覆盖」但更新日志写「增至 129 个」。
- **修复**：统一为 1.1.0、129 个用例。

### 🟢 轻微

#### 10. 可访问性不足
- **位置**：`index.html`
- **问题**：Toast 无 `aria-live`，图标按钮无 `aria-label`，屏幕阅读器用户无法感知提示与操作。
- **修复**：Toast 增加 `role="status" aria-live="polite"`，关闭按钮与搜索框补充 `aria-label`。

## 三、基于 mimo 反馈的视觉优化

mimo 对回收站视图评分 **6.5/10**，指出：对比度低、卡片阴影弱、层次感不足。据此追加视觉优化（保持苹果白克制风格）：

- **禁用搜索框对比度**：新增 `.search-box input:disabled` 样式，禁用态背景改为浅灰、placeholder 提升至 `--text-secondary` 提升可读性。
- **便签卡片阴影增强**：`.note-card` 静态阴影由 `0 1px 3px rgba(0,0,0,0.04)` 提升至 `0 2px 8px rgba(0,0,0,0.06)`，悬停提升至 `0 8px 24px rgba(0,0,0,0.1)`，增强卡片与背景的层次分离。
- **回收站行阴影同步增强**：`.trash-row` 静态/悬停阴影同步加强。

> 说明：mimo 提到的「多色高饱和」属于便签颜色标签这一产品功能特性（用户主动标记），予以保留；按钮形态差异属于语义区分（主操作/恢复/危险删除），符合 Apple HIG 语义化配色原则。

## 六、未修复 / 后续建议

- 便签卡片无键盘导航（方向键），仅鼠标可达——建议后续支持方向键 + Enter 打开。
- 删除移入回收站时 Toast 无「撤销」快捷操作——建议后续在 Toast 中提供一键恢复。
- 窗口尺寸/位置不持久化——建议后续记住上次窗口大小与位置。
