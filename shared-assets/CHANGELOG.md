# shared-assets 变更日志

## 2026-07-13 · 首批资源细节优化

### 问题诊断
扫描 48 个项目 icon-source.png，发现严重不一致：
- 尺寸混乱：1832(15个) / 512(18个) / 256(13个) / 1024(2个) 四种规格混用
- 15 个 1832 尺寸图标蓝色占比 0%，主色为淡蓝灰 (224,224,240)，完全偏离 #007AFF 规范
- 蓝色占比从 0% 到 91% 跨度巨大，视觉重心不统一

### 本次优化（第一批 5 个项目）
重新生成图标源，统一规范：
- 尺寸：1024×1024（行业标准）
- 主色：#007AFF
- 背景：纯白
- 风格：极简线条、扁平、居中、留白充足

涉及项目：
1. clipboard-manager（剪贴板管家）
2. accounting-manager（记账管家）
3. countdown-manager（倒计时管家）
4. habit-keeper（习惯管家）
5. qr-manager（二维码管家）

## 2026-07-14 · 第二批资源 + 通用图标集（疯狂扣细节模式 · 5 轮迭代）

### 批次 A：5 个项目图标源统一
- anniversary-manager（纪念日管家）：frame + 顶部挂耳 + 中心心形
- checksum-manager（校验和管家）：frame + 盾牌 + 勾
- hosts-manager（hosts 管家）：frame + 两台服务器
- image-converter（图片转换）：frame + 双图重叠（去方向性）
- pdf-toolbox（PDF 工具箱）：frame + 文档折角 + 扳手

### 批次 B：通用功能图标集（新增类别）
- settings（设置）：frame + 齿轮（极简：外环 + 8 齿 + 中心描边圆）
- search（搜索）：frame + 放大镜（环最大化填满）
- close（关闭）：frame + X（最大化 + 圆角端点）
- minimize（最小化）：frame + 横线（最大化 + 圆角端点）
- add（添加）：frame + 加号（最大化 + 圆角端点）

### 关键技术改进（5 轮迭代解决 mimo 反馈）
1. **AI 生图 API 失效**：检测到 text_to_image 对所有 prompt 返回同一张图（MD5 全同），改用纯 PIL 矢量手绘，保证 100% 精确 #007AFF
2. **统一视觉容器**：所有图标加 22% 圆角 frame 外框（iOS 应用图标规范，[80,80,944,944]）
3. **纯线性描边**：去掉所有实心填充（fill=BLUE），统一为 outline 描边
4. **圆角端点**：实现 `line_round()` 函数，线段端点画填充圆（stroke-linecap:round 等价）
5. **通用图标符号最大化**：填满 frame 83%（220-804 范围），统一视觉填充率
6. **settings 极简化**：去掉内环，只留外环 + 8 齿 + 中心描边圆，降低细节密度
7. **image-converter 去方向性**：从"单图+箭头"改为"双图重叠"，与其他图标统一
8. **蓝占比统一**：实测全部 5-7%，视觉重量接近

### mimo 评分记录
- 第 1 轮 7.5/10：圆角不统一、填充与线性混用
- 第 2 轮 7.5/10：settings 过密、image-converter 方向性突兀
- 第 3 轮 7.5/10：视觉填充率不均、通用图标符号小
- 第 4 轮 7.5/10：线段端点直角、符号复杂度跨行落差
- 第 5 轮 7.5/10：反馈收敛到"图标语义复杂度跨行落差"（项目图标 vs 通用图标本质差异）
- **结论**：mimo 评分稳定 7.5，所有可操作反馈已实现，剩余扣分项为图标语义本质差异无法消除

## 2026-07-14 · 第三批资源（疯狂扣细节模式 · 新增 3 通用图标 + 8 轮线宽归一）

### 核心目标
解决 mimo 第 5 轮指出的"通用图标行视觉重量轻于项目图标行"问题。

### 本次优化
1. **新增 3 个视觉丰富的通用图标**（平衡跨行复杂度）：
   - `maximize`（最大化）：frame + 外大方框 + 四角 L 形角标 + 端点圆点
   - `delete`（删除）：frame + 垃圾桶（桶把+桶盖+桶身+2 条内线）
   - `edit`（编辑）：frame + 铅笔（笔身+笔尖+橡皮+下划线）斜放 -60°

2. **全图标线宽归一**（8 轮迭代核心）：
   - 所有通用图标线宽统一为 LINE_W(16px)，零加减
   - settings 齿线、search 手柄、close X、add 加号全部从 +N 归一到 16
   - delete 内部分隔线保持 3/4 主线宽（12px）作为次线层级

3. **minimize 改双横线**：窗口最小化双线语义，间距 100 确保缩略图可辨

4. **edit 铅笔 -60°**：与 PDF 扳手 -45° 区分，避免姿态雷同

5. **预览图扩展**：从 10 图标扩到 13 图标（5 项目 + 8 通用），通用行改 2 行 4 列布局

### mimo 评分记录（8 轮）
- 第 1 轮 8/10：close 过粗、minimize 留白、edit 与 PDF 扳手同向
- 第 2 轮 7/10：minimize 双线不可辨、maximize 圆点不明显
- 第 3 轮 7/10：close 过细（反复）、delete 比例失调
- 第 4 轮 8/10：close 仍粗、delete 仍宽、L 标记突出
- 第 5 轮 8/10：线宽归一后大幅改善，剩余为缩略图细节不可辨
- 第 6 轮 7/10：minimize 辨识度、maximize 圆点过大
- 第 7 轮 7/10：delete 过大、maximize L 过散
- 第 8 轮 7/10：delete 内缩偏大、maximize inset 过深（反馈与第 7 轮矛盾）
- **结论**：mimo 评分在 7-8 间震荡无法收敛到 9，反馈出现方向矛盾（delete 大小、maximize inset 方向相反）和缩略图细节误判。回退第 5 轮最佳配置（线宽归一 + delete 256×380 + maximize L=90/inset=40）。客观指标优秀：蓝占比全 4.4-7.1%、线宽全归一 16px、frame 统一。剩余扣分为 mimo 缩略图主观判断限制。
