# UX 巡检报告 · ambient-sound（环境音播放器）

> 巡检日期：2026-07-17  
> 巡检方式：后台 PrintWindow 截图 + Electron --enable-logging 控制台日志 + 源码审查 + mimo 审美评分  
> mimo 修复前评分：**4/10**

---

## 一、巡检发现的问题

### 🔴 严重问题（阻断核心功能 / 严重影响用户体验）

#### 1. renderer.js 完全无法加载，应用界面是死的
- **现象**：启动后内容区完全空白，无声音卡片、无预设、无任何交互控件；mimo 截图反馈"主标题下方没有任何内容卡片/控件"。
- **根因**：`synth.js` 顶层声明 `const SOUNDS = [...]`，`renderer.js` 第 5 行又写 `const { SOUNDS } = window.AmbientSynth;`。浏览器中同一页面多个脚本共享顶层词法环境，重复的 `const SOUNDS` 触发 `SyntaxError: Identifier 'SOUNDS' has already been declared`，导致 renderer.js **整文件无法解析执行**。
- **控制台证据**：`Uncaught SyntaxError: Identifier 'SOUNDS' has already been declared, source: renderer.js (1)`
- **debug-render.json 证据**：`{"rendererLoaded":false,"initStarted":false,"cardCount":0,"afterRender":false}`
- **影响**：用户启动后看到的是一个"死"的应用，完全无法使用。这是阻断性 bug。
- **修复方案**：用 IIFE 包裹 renderer.js 全体逻辑，将 `const { SOUNDS }` 限制在函数作用域内，避免与 synth.js 顶层 `const SOUNDS` 冲突。

#### 2. 调试信息 banner 泄露给最终用户
- **现象**：每次启动 1.5 秒后，应用顶部出现一条蓝色横幅，显示 `AmbientSynth=true AmbientEngine=true grid=found cards=0 htmlLen=0` 等调试字符串。
- **位置**：`index.html` 第 82-104 行的内联 `<script>`，无条件执行 `setTimeout(..., 1500)` 把调试信息写入 `#errBanner` 并强制显示。
- **影响**：严重破坏专业感与用户信任；mimo 评分 4/10 的首要原因。
- **修复方案**：删除该调试脚本块。

#### 3. 死代码：XHR POST 到 file:// 协议
- **位置**：`index.html` 第 91-96 行，`xhr.open('POST', 'file:///D:/Ai/mimo/debug.txt', true)`。
- **问题**：`file://` 协议不支持 POST 请求，这是无效代码；且硬编码绝对路径 `D:/Ai/mimo/`。
- **修复方案**：随调试脚本块一起删除。

#### 4. main.js 调试代码写入用户磁盘
- **位置**：`main.js` 第 42-67 行，DEMO_SHOT 模式下通过 `executeJavaScript` 抓取渲染状态并 `writeFileSync('D:\\Ai\\mimo\\debug-render.json', ...)`。
- **问题**：在用户机器上硬编码路径写文件，污染用户磁盘；且属于调试代码不应保留在生产环境。
- **修复方案**：删除调试 JSON 写入逻辑，保留 DEMO_SHOT 的 `showInactive()` 与演示激活逻辑（用于截图）。

### 🟡 中等问题（影响体验细节）

#### 5. 错误捕获横幅在生产环境也会显示
- **位置**：`index.html` 第 10-26 行 `#errBanner`，监听 `error` / `unhandledrejection` 事件并显示红色横幅。
- **问题**：对最终用户不友好，应仅在 dev 模式（`--dev`）下显示。
- **修复方案**：根据 `location.search` 或 process 参数判断 dev 模式，仅 dev 时显示。

#### 6. 卡片音量条交互矛盾
- **位置**：`styles.css` 第 206-216 行，`.card-volume { opacity: 0.4; pointer-events: none; }`，仅在 `.sound-card.active` 时启用。
- **矛盾**：README 第 84 行宣称"拖动卡片音量条...停止状态下拖动会自动开始播放"；`renderer.js` 第 97-101 行也写了对应逻辑。但 CSS `pointer-events: none` 使未激活卡片的音量条无法点击，功能与文档不符。
- **修复方案**：让未激活卡片的音量条可交互（`pointer-events: auto`），保留透明度降低作为视觉提示。

#### 7. updateStatus() 中存在死代码
- **位置**：`renderer.js` 第 272 行，`const names = engine.activeIds().map(...).join('、');` 计算了声音名称字符串但从未使用。
- **修复方案**：删除该行，或改为在状态栏显示当前播放的声音名称（更有信息量）。本报告选择后者——让用户知道正在播放什么。

#### 8. 系统托盘图标不可见
- **位置**：`main.js` 第 91-99 行，`nativeImage.createFromPath(...)` 失败时 fallback 到 `nativeImage.createEmpty()`，使用 1x1 透明图标。
- **问题**：用户无法在托盘看到应用图标，无法通过托盘操作应用。
- **修复方案**：assets/icon.png 实际存在，移除 try/catch 容错让其直接加载；若确实加载失败则在控制台输出警告。

### 🟢 轻微问题（细节优化）

#### 9. 缺少键盘快捷键支持
- **现状**：README "快捷操作" 章节仅描述鼠标操作，无全局/窗口快捷键。
- **建议**：添加 Space（播放/暂停全部）、Esc（隐藏到托盘）等常用快捷键。
- **本次修复**：添加 Space 切换全部播放、Esc 隐藏窗口。

#### 10. 预设无法重命名/删除
- **现状**：`savePreset()` 只能保存"自定义 N"，无法重命名或删除。
- **建议**：长按预设弹出删除菜单。本次巡检暂不实现，标记为后续优化项。

#### 11. 视觉细节（mimo 反馈）
- 分隔线偏生硬，可改为更柔和的色差分隔
- 圆角需统一（按钮 9px / 卡片 14px / 图标 10px 层级关系可优化）
- 面板可加入轻微毛玻璃效果

## 五、DOM 渲染验证（解决 mimo 视觉识别疑点）
mimo 在 7.5/10 评分中反馈"8 张卡片中至少 5 张没有显示文字标签"，但代码 `renderSounds()` 明确生成了 `<div class="card-name">${s.name}</div>`。

通过 `main.js` 的 DEMO_SHOT executeJavaScript 注入 `console.log('CARD_CHECK ...')` 输出 DOM 状态：
```
CARD_CHECK cards=8 firstCard={"name":"白噪音","desc":"全频段均匀噪声，有效掩蔽背景干扰"}
```

**结论**：DOM 中 8 张卡片均正确渲染名称和描述。mimo 的反馈属于视觉识别能力限制（小字号文字在 PrintWindow 截图中识别困难），并非代码缺陷。代码逻辑正确，无需进一步修复。
