- 复习模式 1-4 评分：✅ 生效
- 复习模式 Esc 退出：✅ 生效
- 新建卡组 Enter 确认：✅ 生效

## 错误处理
- 空输入：✅ 有提示「正面和背面都不能为空」
- 卡组重名：依赖底层未显式校验（不影响功能，默认允许重名）
- 非法导入文件：✅ try-catch 捕获，提示「导入失败」
- 导入字段缺失：✅ 自动补默认调度字段

## 视觉问题（mimo 评分：6.5/10）
- 问题1：删除卡片使用原生 confirm()，与苹果白自定义 UI 风格冲突
- 问题2：侧栏选中态偏弱，仅浅色背景 + 小圆点，辨识度不足
- 问题3：标签（tag）对比度低，浅蓝底色在白色背景上不醒目
- 问题4：统计卡数字颜色逻辑不一致（due 蓝 / fresh 绿 / 其余默认黑）
- 问题5：复习模式无键盘快捷键提示，1-4 评分、空格翻牌是隐藏功能
- 问题6：评分按钮间隔提示模糊（"稍后"/"下次"/"较久"），缺乏实际天数
- 问题7：进度条首张时宽度为 0%，看起来是空的

## 痛点优先级
1. 🔴 严重：缺少 README.md — 违反项目约定（每个项目必须有 README + 下载链接 + 爱发电）
2. 🔴 严重：缺少 Content Security Policy — Electron 安全警告，暴露用户风险
3. 🔴 严重：删除使用原生 confirm() — 与自定义苹果白 UI 风格冲突，打断沉浸感
4. 🟡 中等：模态框不支持 Esc 关闭 — 违反用户直觉（复习模式支持，普通弹窗不支持）
5. 🟡 中等：卡片模态框不支持 Ctrl+Enter 保存 — 新建卡组有 Enter，卡片没有，不一致
6. 🟡 中等：复习快捷键无可见提示 — 1-4 评分是隐藏功能，新用户无法发现
7. 🟡 中等：评分按钮间隔提示模糊 — "稍后/下次/较久"无法辅助决策
8. 🟡 中等：卡组重命名/删除无 UI 入口 — preload 暴露了 deck.rename/remove 但渲染层无按钮
9. 🟡 中等：侧栏选中态偏弱 — 视觉反馈不明确（mimo 指出）
10. 🟡 中等：标签对比度低 — 浅蓝底色不醒目（mimo 指出）
11. 🟡 中等：统计卡数字颜色逻辑不一致 — mimo 指出
12. 🟢 轻微：进度条首张为 0% — 看起来是空的
13. 🟢 轻微：Toast 长文本显示 1.8s 过短 — 导出路径来不及看完

## 修复记录
- ✅ 已修复 [缺少 README]：新建 README.md，含功能介绍、下载链接、使用方式、爱发电、鸣谢
- ✅ 已修复 [缺少 CSP]：index.html 添加 Content-Security-Policy meta，限制 default-src/script-src 为 self
- ✅ 已修复 [原生 confirm]：deleteCard 改为自定义模态框确认，风格统一，含遮罩点击取消
- ✅ 已修复 [模态框 Esc]：全局 keydown 增加 isModalOpen() 判断，Esc 关闭任意模态框
- ✅ 已修复 [卡片 Ctrl+Enter]：openCardModal 正反面 textarea 增加 Ctrl/Cmd+Enter 快速保存
- ✅ 已修复 [复习快捷键提示]：复习界面增加 .review-hints 区域，显示 空格翻牌 / 1-4 评分 / Esc 退出
- ✅ 已修复 [间隔提示模糊]：新增 estInterval() 函数，基于卡片当前 ef/interval/reps 推算实际天数（天/月/年），评分按钮显示如「Again · 1天 [1]」
- ✅ 已修复 [卡组重命名/删除无入口]：deck-item 增加悬浮操作按钮（重命名 + 删除），stopPropagation 避免误触选中，openDeckRenameModal/openDeckDeleteModal 接通已有 IPC
- ✅ 已修复 [侧栏选中态弱]：active 状态增加 inset 左侧 3px 蓝色边条 + 字重 600，视觉反馈明确
- ✅ 已修复 [标签对比度]：tag 增加 border + font-weight 500
- ✅ 已修复 [统计卡颜色不一致]：.stat-value 统一默认 var(--text)，仅 due/fresh 覆盖强调色
- ✅ 已修复 [进度条 0%]：Math.max(4, ...) 保证最小可见宽度
- ✅ 已修复 [Toast 时长短]：长文本（>40 字符）延长至 3.5s
